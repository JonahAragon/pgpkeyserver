Notice: When you submit a key to this server, either via the web interface or via HKP(S) (in your GPG client), that key will be propagated to every server in the sks-keyserver pool. That key will then be stored on each of those servers in addition to this server. Because the transferring of keys between servers is not managed by a central host, deleting a key is not possible. If a key were deleted from this server manually, other servers in the pool will restore the key in order to maintain a complete collection. This is the servers intended behavior. Therefore, the only way to mark a key as not to be used, is to issue a revocation certificate to the keyserver, which must have been made ahead of time using your private key.
